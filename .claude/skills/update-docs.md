# Skill: Update Docs

Use this skill when documentation needs to be created or updated.

## Documentation Files

- README.md
- ROADMAP.md
- PROGRESS.md
- DECISIONS.md
- CLAUDE.md
- app-specific CLAUDE.md files
- infra-specific CLAUDE.md

## Steps

1. Identify what changed.
2. Update only relevant documentation.
3. Keep documentation practical and current.
4. Prefer commands and examples over vague descriptions.
5. Update PROGRESS.md after meaningful milestones.

## Rules

- Do not duplicate large blocks across docs unnecessarily.
- Do not document features that do not exist yet as completed.
- Clearly separate planned work from implemented work.

## Output

Summarize:

- Docs updated.
- Why they were updated.
- Any docs still missing.
