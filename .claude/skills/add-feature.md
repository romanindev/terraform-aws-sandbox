# Skill: Add Feature

Use this skill when adding a new application feature.

## Steps

1. Read README.md, ROADMAP.md, PROGRESS.md, and relevant CLAUDE.md files.
2. Inspect existing code before creating new files.
3. Identify the smallest useful feature slice.
4. Propose a short implementation plan.
5. Implement backend, frontend, tests, and docs as needed.
6. Keep changes focused.
7. Update PROGRESS.md.

## Rules

- Do not introduce unrelated refactoring.
- Do not create duplicate utilities/components.
- Add tests for meaningful behavior.
- Update documentation if commands, setup, or behavior changes.

## Output

When finished, summarize:

- What changed.
- Files changed.
- Tests added or updated.
- Docs updated.
- Follow-up tasks.
